## 📋 발명의 배경

### 종래 기술의 문제점

1. **교실 중심 교육의 한계**
   - Duolingo, Rosetta Stone: 단순 플래시카드
   - Cake, Lingopie: 영상 시청만, 능동적 산출 부족
   - iTalki, Cambly: 비용 부담, 시간 제약

2. **기존 더빙 앱의 한계**
   - Shadowing 앱: 난이도 조절 없는 단순 반복
   - 결과 피드백 밋밋 ("85점입니다")
   - 한국 문화적 맥락 반영 없음

### 본 발명의 목적

1. YouTube 숏츠 콘텐츠를 활용한 "교육이 숨은 1단계" 학습
2. 4단계 Progressive Blanks로 자연스러운 난이도 상승
3. 가요방식 결과 연출로 재도전 유도
4. 다국어 확장 가능한 데이터 구조

## 📜 청구항

### 【청구항 1】 독립 청구항 · Progressive Dubbing UX

**영상 기반 어학 학습 방법에 있어서,**

(a) 동일한 영상 컨텐츠를 서로 다른 자막 공개 수준으로 4단계 제공하는 단계로서,
- 제1 단계: 원본 자막 전체 공개 (빈칸 비율 0%)
- 제2 단계: 핵심어 1~2개를 빈칸 처리 (빈칸 비율 약 20%)
- 제3 단계: 빈칸 비율 약 50% 확대
- 제4 단계: 빈칸 비율 약 85% 이상 (대부분 빈칸)

(b) 각 단계에서 사용자의 발화를 마이크로 수신하고 실시간 분석하여 점수화하는 단계

(c) 각 단계별 점수가 임계값을 초과할 때 자동으로 다음 단계로 진행시키는 단계

를 포함하는 것을 특징으로 하는 어학 학습 방법.

---

### 【청구항 2】 독립 청구항 · 통합 더빙 엔진

**청구항 1에 있어서, 단계 (b)의 실시간 분석은**

(i) 100ms 간격으로 Web Audio API를 통한 음성 볼륨 측정

(ii) 각 문장 구간별 음성 커버리지 계산 (voiceFrames / totalFrames)

(iii) 카라오케식 자막 업데이트 (done/current/next/upcoming 4단계 상태)

(iv) SVG 기반 원형 게이지 시각화 (6단계 색상 전환)

(v) 열 단계 전환 시 파티클 효과 스폰

을 하나의 통합 루프에서 동시 수행하는 것을 특징으로 하는 방법.

---

### 【청구항 3】 독립 청구항 · 크로스플랫폼 오디오 게이트

**청구항 1 또는 2에 있어서, 학습 시작 전 단일 사용자 터치 이벤트로**

(i) AudioContext.resume() 호출

(ii) SpeechSynthesis API 웜업 (빈 문자열 발화)

(iii) YouTube IFrame Player 음소거 해제

(iv) 플랫폼별 특수 처리:
- iOS: 무음 모드 감지 및 경고 표시
- Galaxy: TTS 10초 간격 pause/resume으로 자동 중단 버그 우회

를 동시에 수행하여 모바일 브라우저의 자동재생 차단 정책을 우회하는 것을 특징으로 하는 방법.

---

### 【청구항 4】 독립 청구항 · 가요방식 결과 연출 ★

**청구항 1 내지 3 중 어느 한 항에 있어서, 학습 완료 후 결과 표시 단계가**

(a) 최종 점수를 0에서부터 증가하는 숫자 카운터 애니메이션으로 표시하되,

- 0~50점 구간: 3점/tick 증가 (빠른 상승)
- 50~80점 구간: 1점/tick 증가 (긴장감 증가)
- 80~100점 구간: 0.5점/tick 증가 (클라이맥스)

(b) 최종 점수에 따라 4개 감정 구간으로 분기:

- 95점 이상: "Legend Zone" → 폭죽 효과 + 공유 유도
- 80~94점: "Satisfaction Zone" → 반짝 효과 + 다음 영상 유도
- 60~79점: "Regret Zone" → 진동 효과 + 근접 보상 메시지
- 60점 미만: "Encouragement Zone" → 부드러운 페이드 + 난이도 하향

(c) Regret Zone 에서는 다음 등급까지의 점수 차이를 동적으로 계산하여 "아쉬운 X점! Y점만 더!" 형태의 메시지를 생성하고, 재도전 버튼에 펄스 애니메이션을 적용하여 재도전을 유도

하는 것을 특징으로 하는 방법.

---

### 【청구항 5】 독립 청구항 · 손글씨 기반 단어 학습 시스템 ★

**청구항 1에 기반한 영상 기반 어학 학습 시스템에 있어서, 손가락 터치 기반 손글씨 학습 기능을 포함하는 것으로서,**

(a) **이중 모드 손글씨 입력부**:

- 영상 재생 화면 내 활성화되는 제1 손글씨 영역 (이하 "가리개")으로서, 학습자가 영상 시청 중 단어/표현을 즉흥적으로 손가락 터치로 기록할 수 있는 반투명 오버레이 캔버스
- 개인 학습 공간 화면 내 활성화되는 제2 손글씨 영역 (이하 "Snap Note")으로서, 저장된 학습 표현을 반복 쓰기로 암기할 수 있는 전용 캔버스
- 두 영역은 동일한 Canvas API 엔진을 공유하되, 목적에 따라 다르게 표시됨

(b) **3초 순환 티저 표시부** (이하 "Snap Review"):

- 저장된 N개의 학습 표현 중 1개만 화면 중앙에 큰 카드로 표시
- 3초마다 자동으로 다음 표현으로 페이드 인/아웃 전환
- 카드에는 3개 버튼만 제공: 일시정지, TTS 재생, 삭제
- 저장된 표현 개수에 관계없이 UI 복잡도 일정 (100개든 1000개든 한 번에 1장)

(c) **일시정지 시 자동 펼침 동작**:

- 사용자가 일시정지 버튼 탭 시, Snap Review 아래에 Snap Note가 아코디언 방식으로 펼쳐짐
- Snap Note에는 손가락 터치로 쓰기 가능한 흰색 캔버스 + 연필/지우개/지우기/저장 도구 제공
- 사용자가 재생 버튼을 누르면 Snap Note가 접히고 3초 순환이 재개됨

(d) **반복 쓰기 숙달 시스템**:

- 사용자가 쓰기를 1회 완료할 때마다 카운터 증가 (10개 도트 인디케이터로 시각화)
- 10회 도달 시 "MASTERED!" 뱃지 애니메이션 표시 + 해당 표현의 mastered 속성을 true로 저장
- 손글씨 이미지는 Base64 PNG로 localStorage에 저장되어 나중에 열람 가능

(e) **이중 인코딩 학습 구조** (Dual Coding Theory 적용):

- 시각 채널: 영상 시청 + 텍스트 읽기
- 청각 채널: 원본 음성 + TTS 재생
- 촉각 채널: 손가락 터치 손글씨
- 운동감각 채널: 쓰기 동작

를 포함하여, 사용자의 다감각 학습 경로를 동시 자극함으로써 기억 잔존율을 향상시키는 것을 특징으로 하는 어학 학습 시스템.

---

### 【종속 청구항 5】 · 5요소 채점

청구항 2에 있어서, 최종 점수는 다음 5개 요소의 곱셈으로 산출되는 것을 특징으로 함:
- voiceRatio × timingMatch × volumeConsistency × completionBonus × difficultyMultiplier × 100

### 【종속 청구항 6】 · AI 레슨 파이프라인

청구항 1에 있어서, YouTube 영상 ID를 입력받아:
- 자막을 자동 추출하고
- 대규모 언어 모델 API를 호출하여 번역 및 CORE/HIGHLIGHT 자동 선정
- SnapTalk 표준 JSON 포맷으로 자동 변환

하는 레슨 자동 생성 파이프라인.

### 【종속 청구항 7】 · Step 2 싱크 측정

청구항 1의 제2 단계에서, 100ms 간격으로 싱크로율을 실시간 측정하고 90% 임계값 도달 시 다음 문장으로 자동 진행하는 방법.

### 【종속 청구항 8】 · 발음 정확도 측정 ★

청구항 2에 있어서, Web Speech API의 SpeechRecognition을 추가로 사용하여:
- 사용자 발화를 텍스트로 변환
- 목표 문장과 Levenshtein 거리 기반 유사도 계산
- 단어 단위 매칭률 산출 (70% 이상 유사 시 매칭 인정)

하여 실제 발음 정확도를 측정하는 방법.

### 【종속 청구항 9】 · 다국어 JSON 구조 ★

청구항 1에 있어서, 문장 데이터는 다음과 같은 확장 가능한 구조로 저장:
```
{
  en: (원본),
  core, highlight, start, end,
  translations: {
    ko: {text, highlight},
    ja: {text, highlight},
    zh: {text, highlight},
    es: {text, highlight}
  }
}
```
누락된 언어는 폴백 체인(fallback chain)을 따라 대체 언어 자동 선택.

### 【종속 청구항 10】 · Progressive Reveal 자막
4단계 상태(done/current/next/upcoming) 카라오케 자막 시스템.

### 【종속 청구항 11】 · 6단계 싱크 링 게이지
원형 SVG 게이지에 6단계 색상 전환 적용.

### 【종속 청구항 12】 · 실시간 파티클 루프
열 단계 전환 감지 시 이모지 파티클 스폰.

### 【종속 청구항 13】 · 적응형 난이도 조절 ★
최근 3회 평균 점수 기반 다음 영상 난이도 자동 추천.

### 【종속 청구항 14】 · Interest Scoring
카테고리별 행동 점수 집계 + 30일 시간 감쇠.

### 【종속 청구항 15】 · Difficulty Curve
beginner 5개 → intermediate 15개 → advanced 기본 커브.

### 【종속 청구항 16】 · Auto-Next 추천
학습 완료 후 3초 카운트다운 + Interest/Adaptive 결합 추천.
